1. Суть Проекта
Нужен инструмент, где пользователь может выбрать товар из каталога, ввести его конкретные размеры (например, длина, ширина, диаметр) и сразу увидеть цену. В конце – скачать готовый PDF с расчетом.

2. Основные Компоненты
А. Каталог и Конструктор Формул (Режим Администратора)
Система должна позволять гибко создавать новые типы товаров "на лету", без программиста.

Карточка товара: Название, Фото.
Конструктор переменных: Администратор сам задает, какие параметры нужно ввести для этого товара.
Пример: Для "Отвода" нужны переменные: Диаметр, Угол. Для "Трубы": Длина, Диаметр(эти вещи суоздаются администратором).
Формула цены: Поле, где админ пишет математическую формулу, используя эти переменные.
Пример: 
(Диаметр * 3.14) * Длина * ЦенаМатериала
.
Б. Экран Расчета (Рабочее место менеджера)
Интерфейс разделен на две части:

Слева – Каталог: Список доступных шаблонов товаров (с поиском или прокруткой).
Справа – Смета (Корзина): Список добавленных в расчет позиций.
Сценарий работы:

Менеджер кликает на товар в каталоге (например, "Прямоугольный воздуховод").
товар появляется в смете.
В смете появляются поля для ввода, соответствующие переменным этого товара (которые задал админ).
Менеджер вводит цифры (например, Ширина=500, Высота=300).
Система мгновенно пересчитывает цену за единицу и общую сумму строки.
Можно менять количество (шт) – сумма пересчитывается.
В. Генерация Документа (PDF)
Кнопка "Скачать КП" (Коммерческое Предложение).
При нажатии генерируется PDF-файл.
Содержание PDF: Таблица со списком всех товаров из сметы.
Колонки: №, Название + параметры (напр. "Отвод, Диаметр: 200"), Кол-во, Цена за ед., Сумма.
Внизу итоговая сумма.
Поддержка русского языка (кириллицы) в файле обязательна.

3. Требования к Интерфейсу (UX)
Минимализм: Ничего лишнего, фокус на скорости ввода данных.
Чистота: Поля ввода должны быть удобными, поддерживать десятичные числа (через точку или запятую).
Наглядность: Цены должны обновляться в реальном времени при изменении любой цифры.
Полная Руссификация: Все надписи, кнопки и заголовки на русском языке.

4. Пример Потока Задач для ИИ
Создать базу данных для хранения шаблонов товаров.
Реализовать интерфейс создания товара с динамическими полями.
Сделать "Умный калькулятор", который читает формулу из базы и считает результат в браузере.
Реализовать генерацию PDF на клиенте с поддержкой кириллических шрифтов.

5. Должна быть возможность добавления проектов. Эти проекты сами по себе будут иметь отдельные корзины товаров и ихними значениями. Тоесть все что говорилось, будет храниться в рамке проекта и КП создается в рамке проекта.

6. Есть так же сущьность 'Заказчики'. Заказчики могут быть добавлены в проект и при создании КП, заказчик будет присутствовать в КП. Тоесть при создании проекта, можно добавить заказчика и при создании КП, заказчик будет присутствовать в КП.

7. Сделай проект полностью на русском языке.
